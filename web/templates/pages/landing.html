How it works

1. Deploy an agent

Single Go binary. Run it inside your network — a VPC, a homelab, behind a NAT. No config file needed.

2. Agent runs checks

Ping, HTTP, TCP, DNS. The hub pushes monitor configs to the agent over WebSocket. The agent reports back heartbeats.

3. Get alerted

Three consecutive failures trigger an incident. You get notified via Discord, Slack, or a webhook. No false alarms.

Features

What you get out of the box.

Agent-based monitoring

The agent runs inside your network, so it can reach things that external monitoring tools can't — internal databases, private APIs, services behind a firewall or NAT.

Live dashboard

Status updates stream over SSE. Sparkline charts show latency trends. No refresh needed.

Smart alerts

3-strike rule before firing. Supports Discord, Slack, and generic webhooks.

Four check types

Ping, HTTP, TCP, and DNS. Covers most of what you'd want to monitor in a typical infrastructure setup.

[OK] db.internal:5432 .... 2ms
[OK] redis.internal:6379 .. 1ms
[FAIL] vault.internal:8200 .. timeout

Zero config agent

The agent takes a hub URL and an API key. That's it. Monitor configs are pushed from the hub over WebSocket — no config files, no restarts.

Incident tracking

Incidents are created automatically on failure. Acknowledge, resolve, and track time-to-resolution. Full history per monitor.
